# SourceDeck 1.1.0 — Privacy repair (supersedes v1.0.0)

> **Release-blocking defect fix.** v1.0.0 shipped with developer-authored
> content (contact lists, brand defaults, signatures, social URLs) baked
> into the packaged source. v1.0.0 has been withdrawn. All users on
> v1.0.0 should upgrade to v1.1.0 immediately.

## Downloads

- **macOS (Apple Silicon / M-series):** `SourceDeck-1.1.0-arm64.dmg`
- **macOS (Intel x64):** `SourceDeck-1.1.0-x64.dmg`
- **Windows (x64):** `SourceDeck-Setup-1.1.0.exe`

## What changed

### Privacy
- Shipped source now contains zero owner-identifying data. Empty
  `MOCK_LEADS` array, empty `PROMPT_LIBRARY`, neutral brand default.
  Window title and brand header render as "SourceDeck" on first run.
- Socials tab renders an empty-state card on first run instead of
  hardcoded owner handles. Users configure their own channels in
  Settings.
- Cold-email prompts, GovCon analysis prompts, delivery/task owner
  defaults, and calendar booking links all now pull from
  user-configured brand. Everything starts blank.
- Known leaked contact data removed: 10 real property-management
  companies + real email addresses + real phone numbers.

### First-run safety
- `main.js` now runs a first-run privacy scrub before the window
  is created. If legacy owner state ever lands on a user's machine
  (corrupted prior install, bad migration), it is dropped before the
  UI paints. The scrub only removes; it never seeds.
- Demo / sample data has been moved to `demo/fixtures.json` and is
  **excluded from the packaged app** via `package.json` `build.files`.
  There is no automatic demo hydration.

### Release-blocking checks
- `npm run release:check` now scans every shipped file against a
  30+ pattern blocklist and runs 3 structural checks (MOCK_LEADS
  empty, PROMPT_LIBRARY empty, brand default neutral).
- The GitHub Actions workflow now runs the privacy gate + test
  suite on Ubuntu before any macOS/Windows build can start, and
  re-runs the gate on each build host as defense-in-depth. A
  regression cannot ship.

### Internal
- Clinical Capability manifest provider changed from "ARCG Systems" to
  "SourceDeck"; the corresponding test assertion was updated.
- Legacy Airtable field label `"ARCG Best Services"` is still read
  (for users upgrading from v1.0.0) but all outbound writes now use
  the neutral label `"Recommended Services"`.
- Release notes for v1.0.0 carry a withdrawn/superseded banner.

## Install

- **macOS:** mount the DMG, drag SourceDeck.app to /Applications. First
  launch: right-click → Open (unsigned build).
- **Windows:** run the NSIS installer.

## Upgrading from v1.0.0

The privacy scrub in v1.1.0 will detect owner-identifying strings in
any stored electron-store snapshot and remove contaminated keys on
first launch. Your API credentials (stored separately via
`safeStorage`) are preserved.
