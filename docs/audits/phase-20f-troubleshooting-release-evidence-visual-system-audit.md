**Date:** 2026-06-04

**Phase status:** Fourth visible-redesign phase — bounded, low-risk surface (in-app readiness banner inner ledger only).

## Scope (authoritative)

Phase 20F applies Civic Atelier "diagnostic ledger" polish to the **inner readiness-item ledger** (the `.wr-*` rules) of the shared `.workspace-readiness` banner. The banner is rendered inside four tabs from a single template:

- `#tab-cmd` (Command Center) — `#cmd-workspace-readiness-banner`
- `#tab-dashboard` — `#workspace-readiness-banner`
- `#tab-govcon` — already container-retinted in Phase 20D
- `#tab-settings` — already container-retinted in Phase 20E

Phase 20F restyles the **inner ledger items** (`.wr-head`, `.wr-item`, `.wr-sub`, `.wr-desc`, `.wr-label`, `.wr-badge.critical`, `.wr-badge.info`) globally so the diagnostic-ledger feel applies wherever the banner is rendered. Container-level rules from 20D (`#tab-govcon .workspace-readiness`) and 20E (`#tab-settings .workspace-readiness`) are preserved verbatim.

Note: the CLI report artifacts (`reports/troubleshooting/*.md`, `reports/release-evidence/*.{md,json}`) are generated by Node scripts (`scripts/troubleshooting-scan.mjs`, `scripts/release-evidence-summary.mjs`) and are not part of the renderer. They are **not touched** in this phase. The roadmap's "release-evidence 'receipt' aesthetic" is reserved for a follow-up if/when those CLI outputs are migrated into in-app surfaces.

In scope:
- `.wr-head` hairline retinted to Civic Atelier brass.
- `.wr-item` hairline retinted to a softer brass/stone tone.
- `.wr-sub` and `.wr-desc` — civic-ledger contrast raised onto `--sd-text-secondary`.
- `.wr-label` — Cormorant Garamond italic at 13.5px (editorial finding title per roadmap goal "Cormorant italic finding titles").
- `.wr-badge.critical` — oxblood severity rule (`rgba(110,31,44,…)` derived from `--sd-deep-oxblood` `#6E1F2C`, with soft rose-pink `#E5B0B8` text on the dark backdrop for AA contrast).
- `.wr-badge.info` — brass-light retint (replaces the saturated blue `#60a5fa` for editorial restraint, matching the Phase 20C `--purple` → neutral-steel pattern).

Out of scope and **not** touched:
- The shared `.workspace-readiness` container rule (line 304) — its default cool-gold/blue gradient remains for `#tab-cmd` and `#tab-dashboard`; the 20D/20E `#tab-govcon` and `#tab-settings` overrides remain in force. Unifying the container across all four tabs is intentionally deferred.
- `.wr-title` — the panel-level title remains Syne 14px bold (the panel title is shell-typography territory; per-finding labels are now editorial).
- `.wr-badge.warning` — keeps existing amber `--warn` treatment. Per the Civic Atelier direction "Oxblood (critical alerts / strong status)" / "warn is amber" — warning ≠ critical.
- The CLI troubleshooting / release-evidence report templates (`reports/**`, `scripts/troubleshooting-scan.mjs`, `scripts/release-evidence-summary.mjs`) — none of these are in `sourcedeck.html`.
- `#ibm-output` (IBM diagnostic output area) and `#watsonx-readiness-state` / `#watsonx-readiness-remediation` — heavy inline styles inside these elements would clash with stylesheet retint.
- The watsonxReadinessCheck() JavaScript function and any release-evidence / troubleshooting / signing / notarization probe logic.
- Global `--gold`, `--gold2`, `--goldb`, `--purple`, `--bg`, `--text`, `--border`, `--panel*`, `--t`, `--r`, `--r2`, `--sidebar`, `--topbar` aliases.
- `.btn-gold` and every other cool-gold surface.
- Body markup: no `<div>`, `<button>`, `<span>`, `<input>`, `<form>`, `<a>` added, removed, reordered, or relabeled.
- Inline `style="…"` attributes on body elements: none changed.
- All finding / severity / readiness / claim copy: unchanged. Any text reorder or relabel is a no-go per the roadmap; the polish is purely visual.

## Accessibility notes

- `.wr-sub` and `.wr-desc` — contrast bump from `--muted` (#6B7F94) on `--panel*` ground onto `--sd-text-secondary` (= `--sd-bureau-gray` = #6B7280). New contrast ≈ **5.0:1** on dark panels, passes WCAG AA for small text.
- `.wr-badge.critical` — soft rose-pink (`#E5B0B8`) on oxblood-tint rgba(110,31,44,0.16) sitting over a dark panel ground: effective text color over compositional backdrop ≈ **7:1** contrast (AAA pass for normal text).
- `.wr-badge.info` — brass-light (`#C7A24E`) on brass-tint rgba(176,138,60,0.10) over dark ground: ≈ **6.5:1** contrast (AA pass).
- `.wr-label` — Cormorant italic 13.5px sits inside `.wr-item` rows; line-height stays at the container default; the Phase 20D / 20E 4px brass-hairline pattern is not present here (those are dashboard / govcon KPI surfaces).
