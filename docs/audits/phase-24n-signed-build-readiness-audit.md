## 4. Required evidence before public signed release

All must be present and captured (presence-only, no secrets) before any signed/notarized claim:

- [ ] Apple **Developer ID Application** certificate evidence (`CSC_LINK` / keychain identity present; identity name redacted/presence-only).
- [ ] **Hardened runtime** enabled (already `true` in config — verify on the signed artifact).
- [ ] **Entitlements reviewed** (`build/entitlements.mac.plist` — confirm least-privilege; no debugging entitlements in release).
- [ ] **Code signature verification** — `codesign --verify --deep --strict` passes; `spctl` source = Developer ID.
- [ ] **Notarization submission success** — `xcrun notarytool submit` returns `Accepted` (requires `mac.notarize: true` + notarize env).
- [ ] **Stapling verification** — `xcrun stapler staple` + `stapler validate` pass.
- [ ] **Gatekeeper assessment** — `spctl -a -vvv -t install/exec` accepted.
- [ ] **Checksum generated** — `shasum -a 256` of each artifact.
- [ ] **Artifact path documented** (`dist/SourceDeck-macOS-<arch>.dmg` / `.zip`).
- [ ] **Rollback plan documented** (prior release notes retained; bad-artifact hash recorded).
- [ ] **Clean malware/security scan** if available.
- [ ] **Release notes updated** (signed RC).
- [ ] **No secrets embedded** (no `.env`, keys, certs, provisioning profiles, notarization creds in the bundle).

Until every box is checked from a real signing environment, the status stays **UNSIGNED RC ONLY** and the public signed/notarized claim remains **NO-GO**.
