# SourceDeck Lemon Squeezy Secure Fulfillment Plan

## Executive Decision

SourceDeck should use **Lemon Squeezy Checkout and license keys for the first commercial release**, with a lightweight secure download layer in front of release artifacts. Lemon Squeezy is the fastest professional path because it can generate license keys and send webhooks after order and license events, while SourceDeck can validate customer licenses inside the desktop app through the Lemon Squeezy License API.[^ls-license-api] [^ls-webhooks]

The operating rule is simple: **payment creates the license, the license controls app use, and the download link should be time-bound rather than manually emailed**. The first release can use a private delivery URL generated by a fulfillment service, while later releases can graduate to a full customer portal.

## Target Flow

| Stage | System | Outcome |
|---|---|---|
| Checkout | Lemon Squeezy hosted checkout | Customer pays for SourceDeck and Lemon Squeezy issues the license key. |
| Webhook | SourceDeck fulfillment endpoint | The purchase event is verified and recorded without exposing secrets in the desktop app. |
| License email | Lemon Squeezy email receipt | Customer receives the license key generated by the product license policy. |
| Secure download | Cloudflare Worker plus R2 or private release artifact | Customer receives a time-bound signed download URL for the current signed/notarized `.dmg`. |
| First launch | SourceDeck desktop app | Customer enters the license key in Settings and activates against Lemon Squeezy. |
| Ongoing validation | SourceDeck desktop app | The app validates the license periodically and uses offline grace if the network is temporarily unavailable. |

## Lemon Squeezy Product Configuration

The SourceDeck product should be configured as a **software product with licensing enabled**. The license policy should start strict enough to protect the product, but not so strict that normal customer onboarding becomes a support problem.

| Setting | Recommended v1 Value | Reason |
|---|---|---|
| Product name | `SourceDeck GovCon Professional` | Clear customer-facing product name. |
| Billing model | Annual subscription or annual license | Aligns with B2B operational software value and support cost. |
| License activation limit | 1 for Solo, 3 for Team | Prevents casual sharing while supporting normal buyer use. |
| License validity | Matches subscription term | Keeps entitlement aligned with payment status. |
| Refund/cancellation behavior | Disable or downgrade after grace period | Protects access while avoiding abrupt customer lockouts from payment delays. |
| Checkout success redirect | SourceDeck download/activation instructions page | Gives customers one controlled next step. |
| Receipt content | Include license key and install instructions | Prevents support tickets after purchase. |

## Fulfillment Endpoint Contract

A minimal fulfillment endpoint should receive Lemon Squeezy webhook events, verify the webhook signature, and create a short-lived download token for eligible purchases. The endpoint should never expose Apple credentials, Lemon Squeezy API keys, webhook secrets, GitHub tokens, or storage signing secrets to the desktop app.

| Event | Required Behavior |
|---|---|
| `order_created` or paid order event | Confirm the product/variant maps to SourceDeck and create a customer fulfillment record. |
| `license_key_created` or license event | Attach the generated license key metadata to the customer record if available. |
| Refund/cancellation event | Mark entitlement as restricted so future downloads and app validation can be blocked or downgraded. |
| Manual resend request | Generate a new short-lived download link only for a verified paid customer/license. |

## Secure Download Strategy

For v1, the most efficient A+ approach is **Cloudflare Worker + R2 private object storage**. The signed/notarized `.dmg`, `.zip`, checksum file, and release notes are uploaded to private R2 storage. The Worker validates a fulfillment token and issues a short-lived download response. This is stronger than a public static URL and simpler than building a full account portal immediately.

| Option | Use Now? | Notes |
|---|---:|---|
| Lemon Squeezy hosted file delivery | No, except as fallback | Fast, but less control over versioning, evidence, and future updates. |
| Public GitHub Release artifact | No for paid customer download | Acceptable for open downloads if license gates usage, but weaker as paid delivery. |
| Private GitHub Release artifact with tokenized proxy | Possible | More moving pieces and token scope risk. |
| Cloudflare R2 private object with Worker gate | **Yes** | Best blend of control, security, speed, and low operating overhead. |
| Full customer portal | Later | Better for enterprise accounts, invoices, seats, and renewals, but slower for v1. |

## Required Environment Variables

These variables belong in the fulfillment service or CI secret store. They must not be committed to the repository.

| Variable | Location | Purpose |
|---|---|---|
| `LEMONSQUEEZY_WEBHOOK_SECRET` | Fulfillment service | Verifies incoming webhook authenticity. |
| `LEMONSQUEEZY_API_KEY` | Fulfillment service only if server-side API lookups are needed | Retrieves order/license details when webhook payload is insufficient. |
| `SOURCEDECK_PRODUCT_ID` | Fulfillment service | Confirms webhook belongs to SourceDeck. |
| `SOURCEDECK_VARIANT_IDS` | Fulfillment service | Maps Solo/Team/Deployment entitlements. |
| `DOWNLOAD_SIGNING_SECRET` | Fulfillment service | Signs and verifies short-lived download tokens. |
| `R2_BUCKET` | Cloudflare Worker binding | Stores signed/notarized artifacts privately. |
| `SOURCEDECK_CURRENT_MAC_DMG_KEY` | Fulfillment service | Points to the current customer release artifact. |
| `SOURCEDECK_CURRENT_MAC_SHA256` | Fulfillment service | Shows customers the expected artifact checksum. |

## Customer Email Copy Requirements

The post-purchase email should be short, controlled, and support-resistant. It should include the license key, current version, download link, checksum link, and support contact. It should not include internal build logs, GitHub tokens, or raw storage URLs.

| Email Field | Required Content |
|---|---|
| Subject | `Your SourceDeck license and secure download` |
| License key | Lemon Squeezy-generated key. |
| Download link | Short-lived fulfillment URL, not a permanent storage URL. |
| Install note | `Open the DMG, drag SourceDeck to Applications, then activate with your license key.` |
| Security note | `The app is signed and notarized for macOS distribution outside the App Store.` |
| Support | ARCG/SourceDeck support mailbox. |

## Acceptance Tests

A paid SourceDeck release is not ready until these tests pass in test mode and production mode.

| Test | Required Result |
|---|---|
| Test purchase | Lemon Squeezy generates a license key and sends the expected receipt. |
| Webhook verification | Invalid signatures are rejected; valid signatures are accepted. |
| Product mapping | Non-SourceDeck product/variant events are ignored. |
| Secure download | Customer receives a short-lived link and can download the correct `.dmg`. |
| Hash verification | Downloaded artifact SHA-256 matches release evidence. |
| App activation | Valid test license activates SourceDeck. |
| Bad key rejection | Invalid license is rejected without storing a key. |
| Refund/cancel handling | Revoked entitlement can no longer receive new download links. |
| Secret safety | No privileged secret appears in app bundle, renderer, logs, or repository files. |
