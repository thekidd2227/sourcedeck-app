# Phase 25K — Paid Pilot Delivery Posture

**Date:** 2026-06-10
**Companion:** `phase-25k-website-app-parity-audit.md`, `phase-25k-paid-pilot-readiness-gate.md`.

---

## 1. Buyer path (unchanged)

```
Public website → Request Access → Manual Qualification → Approved Onboarding → Pilot Delivery Artifact
```

This matches the Phase 25C master delivery method. **No public checkout. No self-serve signup. No public download.** The buyer's first touchpoint is `/request-access/` on `sourcedeck.app`; the operator qualifies before any delivery artifact is offered.

## 2. Mass-delivery target (future)

The canonical mass-delivery target remains the **secure web app / PWA**. The web/PWA delivery contract is documented in `docs/product/phase-25c-secure-web-pwa-delivery-contract.md`. Phase 25K does not ship that surface.

## 3. Current paid-pilot delivery artifacts (today)

The operator chooses **exactly one** of the following for each paid-pilot buyer, in writing, before delivery:

### Option A — Web/PWA pilot access

- Buyer receives a URL to a tenant-scoped hosted instance.
- Authentication required on every session.
- No anonymous access.
- Onboarding mirrors the Phase 24K Setup Wizard.
- Phase 25H Calendar, Phase 25E Proposal Workspace, Phase 25F GovCon Capture OS surfaces are reachable.
- Phase 25I FAR Reference + Word/PDF export are **not** available until Phase 25I recovery PR merges to `main`.
- Delivered via the hosted web app once that infrastructure ships (target: post-recovery).

**Status:** Documented; not currently shipped. Reserve for the day the web/PWA infrastructure is operational.

### Option B — Controlled desktop RC pilot

- Buyer receives the unsigned dev/RC desktop bundle through the buyer's secure transfer channel.
- Operator walks the buyer through macOS Gatekeeper "open anyway" on first launch.
- Buyer is told (in writing) that the build is an **unsigned development RC**.
- No public download link is posted.
- The Phase 25B 7-day internal trial framework adapts directly: same Day 0 → Day 7 checklist, same go/no-go scorecard, same troubleshooting log template.
- Phase 25I FAR Reference + Word/PDF export are **not** available until the recovery PR merges and the bundle is rebuilt.

**Status:** Documented; available today for direct-relationship pilots. This is the default for the current pilot window.

## 4. What is explicitly NOT a paid-pilot delivery artifact

- ❌ The Desktop ZIP package generated by the Phase 25B Day 0 refresh script (`bash ~/sd-day0-refresh.sh`). That is **internal trial only** — used by the operator to validate the product on their own Mac. It is **never** sent to a buyer.
- ❌ A public GitHub Release. Phase 25A invariant: no GitHub Release for the unsigned RC.
- ❌ A public S3 or Dropbox link to the bundle.
- ❌ A public download link on `sourcedeck.app` (`/download/app/` route remains unlinked from nav per Phase 25C).
- ❌ A `Free demo` / `Try now` / `Get started free` CTA on the website (Phase 25C invariant).

## 6. Per-customer delivery letter template (operator action)

When the operator initiates a paid-pilot engagement, the letter must:

1. Name the chosen delivery artifact (Option A or Option B from §3).
2. Cite the Phase 25B 7-day internal trial framework as the buyer's first 7 days.
3. State explicitly that the build is an **unsigned development RC** (for Option B) or a **tenant-scoped hosted instance** (for Option A).
4. Reference the Phase 25A pilot launch plan §3 (no-send / no-submit / no-upload posture).
5. Reference the Phase 25C delivery boundary docs.
6. Cite this Phase 25K readiness gate document as proof the operator has verified buyer-facing claims match what ships on `main`.
7. Be countersigned by the operator AND ARCG before delivery.

A draft letter template is reserved for a future phase (provisionally Phase 25L) once the operator surfaces the first pilot buyer.
